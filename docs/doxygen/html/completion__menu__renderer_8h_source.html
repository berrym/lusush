1/**
2 * @file completion_menu_renderer.h
3 * @brief Completion Menu Renderer - Text Formatting from Menu State
4 *
5 * Converts completion menu state (from completion_menu_state.h) into
6 * formatted text ready for display via screen_buffer system.
7 *
8 * Architecture:
9 * - Pure formatting - no state management
10 * - No terminal I/O - outputs text only
11 * - Respects menu state's visible range (scrolling support)
12 * - Supports selection highlighting, category headers, type indicators
13 * - Multi-column layout for efficient screen usage
14 *
15 * Integration:
16 * - Input: lle_completion_menu_state_t (from Phase 4)
17 * - Output: Formatted text string with \n separators
18 * - Consumer: command_layer or display_controller
19 * - Display: screen_buffer_render() handles actual rendering
20 */
21

98/**
99 * Render completion menu to text
100 *
101 * Generates formatted menu text from menu state. Respects visible range
102 * from menu state (first_visible, visible_count) for scrolling support.
103 *
104 * Output format:
105 * ```
106 * completing external command
107 * ls lsb-release lscpu lsiio
108 * lsar lsb_release lsdiff lsinitrd
109 * completing alias
110 * > ls lsa lsd lsl
111 * ^^^ (selected item, highlighted)
112 * ```
113 *
114 * Features:
115 * - Multi-column layout (fits more items per row)
116 * - Category headers (bold text)
117 * - Selection highlighting (reverse video or prefix marker)
118 * - Type indicators (if enabled)
119 * - Scrolling support (renders only visible range)
120 *
121 * @param state Menu state (contains results, selection, visible range)
122 * @param options Rendering options (NULL for defaults)
123 * @param output Buffer for rendered menu text
124 * @param output_size Size of output buffer
125 * @param stats Optional statistics output (can be NULL)
126 * @return LLE_SUCCESS or error code
127 */
