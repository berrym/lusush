Detailed Description

LLE Transient Prompt System - Type Definitions.

Specification: Spec 25 Section 12 - Transient Prompt System Version: 1.1.0

The transient prompt system simplifies previous prompts in the terminal scrollback after commands execute, reducing visual clutter. When enabled, fancy multi-line prompts are replaced with a minimal transient format after the user presses Enter.

Example: Before: [user@host] ~/path (git-branch *) $ echo hello After: ‚ùØ echo hello

Implementation: The transient replacement is triggered by the LLE_HOOK_LINE_ACCEPTED widget hook, which fires after the user presses Enter but before dc_finalize_input(). The actual rendering is done through dc_apply_transient_prompt() in the display controller, ensuring proper screen buffer state management.

This header provides type definitions used by the prompt composer. The implementation lives in the widget system (builtin_widgets.c) and display controller (display_controller.c).
