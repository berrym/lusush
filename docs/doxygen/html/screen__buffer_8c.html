Detailed Description

Screen Buffer Management Implementation.

Virtual screen buffer system for reliable terminal updates. Based on techniques from ZLE (zsh), Fish, and Replxx.

Architecture:

  1. Maintain a virtual representation of what's on the terminal screen
  2. When content changes, render new state into a new virtual screen
  3. Diff old vs new virtual screens to find what changed
  4. Generate minimal escape sequences to apply only the changes

This solves line wrapping issues because we never rely on cursor movement tricks or assumptions about terminal state.
